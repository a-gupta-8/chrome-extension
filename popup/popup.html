Arrays

Arrays are one of the most common data structures in computer science. Arrays are stored contiguously in memory, which means blocks of memory are allocated for them and there can be no gaps between the memory addresses of its values.

The elements in an array can be identified by their index which represents thier position in the array. Usually, the first element has an index 0, the second element has an index 1 and so on.

If an array extends past the memory block it's allocated, then a new array, of larger size, is created and the the values are copied over to the next array.

Similarly, in some cases, if the array becomes too small then a smaller sized array is allocated and the values copied over

Time Complexity (amortised)
Insert at beginning O(n)
Insert at end O(1)
Remove from beginning O(n)
Remove from end O(1)
Access index O(1) 
        
          # In Python

          myArray = []
          myArray.append(5)
          myArray[0]
          myArray.pop()

Linked Lists

Linked Lists are another way of storing lists of data. The difference between Linked Lists and Arrays are that linked lists do not have indexing, so you cannot access elements by their index, because they are not stored contiguosly in memory.

Instead, linked lists use pointers to point to their next value. Each element in the linked list is stored as a node which consists of the value itself and a pointer to the next node.

The above shows a singly-linked list where the each node points to the next node, but there can also be doubly linked lists where each node has an additional "previous" pointer that points to the previous node.

Time Complexity (amortised)
Insertion O(1)
Remove element O(1)
Access element O(n) 
        
          # In Python

          class Node(element):
            val = value
            next = None
          
          head = Node(5)
          head.next = Node(1)

Hash Sets

Hash Sets is a typical set but a little bit better. A set cannot have duplicate values which is also the case in hash sets, but hash sets use a hashing function which allows for constant time lookup of values.

A hashing function pretty much takes in the element as an input and does some sort of arithmic calculation on it to produce a hash value/id. Then some modulus of that hash value is used as the index of the value to store at.

A good hashing function is one that produces different hash values for different inputs. But in some cases two inputs can have the same hash value, in which case there are two methods used when inserting those values.

1. Insert the value in the next available address

2. Make a linked list at the point and append the value as a node

Time Complexity (amortised)
Insertion O(1)
Remove element O(1)
Access element O(1)

Hash Maps

Hashmaps are very similar to hash sets but instead of just the value being stored, they store a (key, value) pair where you can access the values by user defined keys.

The benefit here is that, we are not restricted to using indexes/numbers for accessing values, rather we can define how we want to acess those values.

Hash maps are also called objects in some languages and can be used to store multiple data types into one data type.

Time Complexity (amortised)
Insertion O(1)
Remove element O(1)
Access element O(1)

Graphs

Graphs are used as a way to structure relationships between elements. Graphs can be directed or undirected and can be stored as an adjacency matrix or adjacency list. Graphs usually are accessed through traversal algorithms such as BFS and DFS. Similar to other data structures, each element in a graph can be stored as a node.
